# Requirements 文件夹

该文件夹用于存放项目的需求规格说明书。

## 文件格式

支持以下文件格式：
- `.md` - Markdown格式（推荐）
- `.txt` - 纯文本格式
- `.docx` - Word文档格式

## 需求文档模板

每个功能模块应创建一个独立的需求文档，建议使用以下模板：

```markdown
# [功能模块名称]需求规格说明书

## 功能描述
简要描述该功能模块的用途和目的

## 输入条件
- 参数1：说明
- 参数2：说明
- ...

## 预期结果
- 正常情况：预期行为
- 异常情况1：预期提示
- 异常情况2：预期提示
- ...

## 边界条件
- 边界值1
- 边界值2
- ...

## 业务规则
- 规则1
- 规则2
- ...
```

## 示例

参考 `login_requirements.md` 文件了解完整的需求文档示例。

## 测试用例输出格式

系统将根据需求文档生成包含以下八大字段的测试用例表格：

| 字段 | 说明 | 示例 |
|------|------|------|
| 用例编号 | 唯一标识，格式为 TC-模块名-序号 | TC-LOGIN-001 |
| 用例标题 | 简洁描述测试内容 | 测试登录成功 |
| 模块 | 所属功能模块 | 用户登录 |
| 优先级 | P0/P1/P2（P0最高） | P0 |
| 前置条件 | 执行测试前需满足的条件 | 用户已注册 |
| 测试步骤 | 详细的操作步骤，每条步骤编号 | 1. 打开登录页面<br>2. 输入手机号和密码<br>3. 点击登录按钮 |
| 测试数据 | 测试中使用的具体数据 | 手机号：13800001001<br>密码：Aa123456 |
| 测试结果 | 预期结果，用例未执行时填"待测" | 登录成功，显示用户手机号 |

## 使用方法

1. 将需求规格说明书放入此文件夹
2. 在项目根目录下执行"生成测试用例"命令
3. 系统将自动读取此文件夹下的需求文档并生成Markdown表格格式的测试用例
4. 测试用例文件将保存到 `script/` 文件夹下，命名格式为 `test_XX_模块名.md`
