# 测试用例生成器

## 功能概述

该技能用于根据项目Requirements文件夹下的需求规格说明书自动生成测试用例表格，包含用例编号、用例标题、模块、优先级、前置条件、测试步骤、测试数据、测试结果八大字段。

## 触发条件

当用户提出以下请求时，必须调用此技能：
- "生成测试用例"
- "根据需求生成测试用例"
- "编写测试用例"
- "创建测试用例"

## 使用步骤

### 步骤1：检查需求规格说明书

1. 检查项目根目录下的 `Requirements/` 文件夹
2. 读取文件夹中的所有需求规格说明文件（支持 .md, .txt, .docx 等格式）
3. 分析需求文档，提取功能点和测试场景

### 步骤2：分析需求规格说明

从需求文档中提取以下信息：
- 功能模块名称
- 功能描述
- 输入条件
- 预期结果
- 边界条件
- 异常场景

### 步骤3：生成测试用例

根据分析结果，生成包含以下八大字段的测试用例表格：

| 字段 | 说明 | 示例 |
|------|------|------|
| 用例编号 | 唯一标识，格式为 TC-模块名-序号 | TC-LOGIN-001 |
| 用例标题 | 简洁描述测试内容 | 测试登录成功 |
| 模块 | 所属功能模块 | 用户登录 |
| 优先级 | P0/P1/P2（P0最高） | P0 |
| 前置条件 | 执行测试前需满足的条件 | 用户已注册 |
| 测试步骤 | 详细的操作步骤，每条步骤编号 | 1. 打开登录页面<br>2. 输入手机号和密码<br>3. 点击登录按钮 |
| 测试数据 | 测试中使用的具体数据 | 手机号：13800001001<br>密码：Aa123456 |
| 测试结果 | 预期结果，用例未执行时填"待测" | 登录成功，显示用户手机号 |

#### 测试用例类型

1. **正常场景测试**：验证功能在正常输入下的正确性
2. **边界条件测试**：验证参数边界值的处理
3. **异常场景测试**：验证异常输入的错误提示

#### 优先级定义

- **P0**：核心功能，影响主流程，必须通过
- **P1**：重要功能，影响部分流程，建议通过
- **P2**：一般功能，不影响主流程，可选通过

### 步骤4：输出测试用例

将生成的测试用例以Markdown表格形式输出，并保存到项目的 `script/` 文件夹下的对应测试文件中。

#### 文件命名规则

- 文件命名格式：`test_XX_模块名.md`
- XX为两位数字序号，从01开始递增
- 示例：`test_01_login.md`, `test_02_register.md`

## 示例

### 需求规格说明示例

```markdown
# 用户登录功能需求

## 功能描述
用户通过手机号和密码登录系统

## 输入条件
- 手机号：11位数字
- 密码：6-16位，包含大小写字母和数字

## 预期结果
- 登录成功：显示用户手机号
- 密码错误：提示"密码错误"
- 用户不存在：提示"用户不存在"
- 密码为空：提示"密码不能为空"
```

## 注意事项

1. 确保需求规格说明书完整清晰
2. 如果Requirements文件夹为空，提示用户先添加需求文档
3. 生成的测试用例必须包含八大字段：用例编号、用例标题、模块、优先级、前置条件、测试步骤、测试数据、测试结果
4. 用例编号格式为 TC-模块名-序号，序号从001开始
5. 优先级分为P0/P1/P2，根据功能重要性合理分配
6. 测试步骤需详细、可执行，每条步骤独立编号
7. 测试数据需具体明确，便于执行和验证
8. 测试结果为预期结果，未执行时填"待测"
9. 如果需求涉及的功能模块没有对应的Page Object类：
   - 检查 `page/` 目录下是否存在对应文件
   - 如果不存在，先创建对应的Page Object类
   - 命名格式：`page_模块名.py`，类名：`Page模块名`
   - 参考项目中已有的Page Object实现风格

## Page Object创建指南

当需求涉及新模块且没有对应Page Object时，按照以下步骤创建：

### Page Object文件结构

```python
from base.base import Base

class Page模块名(Base):

    def __init__(self, driver):
        Base.__init__(self, driver)

    def 操作方法1(self, 参数):
        # 执行操作逻辑
        pass

    def 操作方法2(self):
        # 执行操作逻辑
        pass

    def get_success_result(self):
        # 获取成功结果
        pass

    def get_fail_result(self):
        # 获取失败结果
        pass
```

### 文件命名规则

- 文件命名格式：`page_模块名.py`
- 示例：`page_login.py`, `page_register.py`

### 类命名规则

- 类名格式：`Page模块名`（首字母大写）
- 示例：`PageLogin`, `PageRegister`
